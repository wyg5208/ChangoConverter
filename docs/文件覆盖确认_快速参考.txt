===================================================================
            文件覆盖确认功能 - 快速参考指南 v1.2.0
===================================================================

【功能概述】
当转换的目标文件已存在时，系统会弹出对话框让您选择处理方式

===================================================================
【三种处理方式】
===================================================================

1. 覆盖现有文件 ⚠️
   ├─ 删除原文件
   ├─ 生成新文件（同名）
   ├─ 适用：确认要更新文件
   └─ 日志：⚠️ 将覆盖现有文件

2. 重命名新文件 📝
   ├─ 保留原文件
   ├─ 新文件添加数字后缀（如 file_1.docx）
   ├─ 适用：保留所有版本
   └─ 日志：📝 重命名为: document_1.docx

3. 跳过此文件 ⊘
   ├─ 不转换
   ├─ 原文件不变
   ├─ 适用：不需要转换此文件
   └─ 日志：⊘ 跳过 docx (文件已存在)

===================================================================
【记住选择功能】
===================================================================

☑ 记住我的选择，后续相同问题使用相同方式处理

效果：
├─ 勾选后，后续文件冲突自动应用相同策略
├─ 不再弹出对话框
├─ 仅在当前转换任务有效
└─ 新转换任务会重置策略

===================================================================
【推荐使用场景】
===================================================================

场景1：首次转换
推荐：随意选择（通常不会有冲突）

场景2：更新已有文件
推荐：覆盖 + ☑ 记住选择
效果：批量更新所有文件

场景3：保留版本对比
推荐：重命名 + ☑ 记住选择
效果：生成 file_1.docx, file_2.docx...

场景4：增量转换（只转换新文件）
推荐：跳过 + ☑ 记住选择
效果：只转换不存在的文件，跳过已存在的

场景5：精细控制每个文件
推荐：不勾选记住选择
效果：每个文件单独决定

===================================================================
【重命名规则】
===================================================================

原文件         →  新文件名
document.docx  →  document_1.docx
document_1     →  document_2.docx
document_2     →  document_3.docx
...

自动查找可用的数字后缀

【注意事项】
===================================================================

⚠️ 覆盖操作不可逆
   └─ 建议重要文件先备份

⚠️ 策略会在新转换任务时重置
   └─ 确保每次都能重新选择

⚠️ 重命名会产生多个版本文件
   └─ 需要定期清理

⚠️ 跳过后该文件不会被转换
   └─ 误选需重新开始转换

===================================================================
【快速操作指南】
===================================================================

步骤1：选择输入文件/文件夹
步骤2：选择输出格式
步骤3：点击"▶ 开始转换"
步骤4：如果文件已存在，弹出对话框
步骤5：选择处理方式（覆盖/重命名/跳过）
步骤6：决定是否勾选"记住选择"
步骤7：点击"确定"继续转换

批量转换建议：
└─ 第一个文件时决定整体策略
└─ ☑ 勾选"记住选择"
└─ 后续文件自动应用相同策略
