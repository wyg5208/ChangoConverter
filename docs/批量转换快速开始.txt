✨ 新功能：批量转换模式
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

现在支持选择一个文件夹，自动扫描所有子文件夹中的文档并批量转换！

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 5步快速开始
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣  启动程序
    运行 app_all_function.py 或打包的可执行文件

2️⃣  选择批量转换模式
    ○ 单文件转换
    ● 批量转换（文件夹）  ← 选这个

3️⃣  选择文件夹
    点击 [浏览] 按钮，选择包含文档的文件夹
    
    提示：程序会自动扫描所有子文件夹！

4️⃣  选择输出格式
    勾选需要的格式（可多选）：
    ☑ HTML 5
    ☑ Microsoft Word (DOCX)
    ☑ PDF (需要LaTeX)
    ☑ Plain Text
    ... 还有40+种格式可选

5️⃣  开始转换
    点击 [开始转换] 按钮
    
    观察日志区域的实时进度！

场景1：将博客文章批量转为HTML
────────────────────────────────────
    文件夹结构：
    Blog/
    ├── 2024/
    │   ├── article1.md
    │   └── article2.md
    └── 2023/
        └── old-post.md
    
    操作：
    1. 选择批量转换模式
    2. 选择 Blog 文件夹
    3. 勾选 HTML 5
    4. 开始转换
    
    结果：
    Blog/
    ├── 2024/
    │   ├── article1.md
    │   ├── article1.html  ← 新生成
    │   ├── article2.md
    │   └── article2.html  ← 新生成
    └── 2023/
        ├── old-post.md
        └── old-post.html  ← 新生成

场景2：文档库多格式输出
────────────────────────────────────
    文件夹：D:/Documents/
    
    操作：
    1. 选择批量转换模式
    2. 选择 D:/Documents/ 文件夹
    3. 勾选多个格式：
       ☑ HTML 5
       ☑ DOCX
       ☑ PDF
       ☑ Plain Text
    4. 开始转换
    
    结果：每个文档都生成4个版本！

💡 重要提示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✓ 支持的格式
  • Markdown (.md, .markdown)
  • Word (.docx)
  • HTML (.html, .htm)
  • LaTeX (.tex)
  • 等30+种输入格式

✓ 自动功能
  • 递归扫描所有子文件夹
  • 自动识别文件格式
  • 自动保存到源文件文件夹
  • 跳过与源文件相同的格式

✓ PDF转换
  • 需要先安装 LaTeX
  • Windows推荐：MiKTeX (https://miktex.org/)
  • 首次转换会自动下载必要组件

⚠ 注意事项
  • 转换会覆盖已存在的同名文件
  • 建议先用小文件夹测试
  • 确保有足够的磁盘空间

Q: 能自定义输出文件夹吗？
A: 批量模式自动保存到源文件夹，保持目录结构。
   如需自定义位置，请使用"单文件转换"模式。

Q: 会覆盖源文件吗？
A: 不会。只有当输出格式与输入格式完全相同时才会跳过。

Q: 支持多深的子文件夹？
A: 无限制！支持任意深度的目录结构。

Q: PDF转换失败？
A: 需要先安装LaTeX。
   Windows用户推荐下载安装 MiKTeX。
